# Progressive context delivery

Continuum uses `minimum-safe-progressive-v1`. The first delivery prioritizes required implementation, tests, public contracts, and safety constraints. Later deltas are driven by explicit requests, test failures, missing coverage, or out-of-scope modifications.

Already-active versions are returned as references without full content. Restored, omitted, and newly supplied items remain distinct in output and persistence. Completion expires active presence records.

Context token counts and duplicate-token avoidance are deterministic estimates. They are not provider billing measurements and do not establish an optimal packet or causal task improvement.
