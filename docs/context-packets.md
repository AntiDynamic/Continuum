# Context Packets

A packet separates a small orientation section from implementation material and metadata-only escalation candidates. Orientation covers repository shape, architecture/safety constraints, risk, and state. Implementation contains exact symbols, contracts, tests, configuration, schema, and safety evidence.

The default budget is 8,000 estimated tokens, 25 items, and 1,800 tokens per item, with orientation and required-coverage reserves. Selection favors mandatory correctness/safety, exact targets, tests, contracts, verified relationships, and architecture. Equal hashes, normalized duplicates, excessive same-file items, and oversized items are omitted with reasons. Escalation candidates never contain full content.

Packets are deterministic, budget-constrained ranked context—not guaranteed minimum or optimal context.
